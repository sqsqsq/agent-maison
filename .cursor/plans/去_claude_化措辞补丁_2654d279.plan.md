# 去 claude 化措辞补丁

## 背景与判断

| Layer | generic 是否覆盖 | 处理 |
|---|---|---|
| Layer 1 提示加固 | ✅（共用 [framework/templates/AGENTS.md.template](framework/templates/AGENTS.md.template)） | 模板已中立，无需动 |
| Layer 2 完成回执 | ✅（共用 [framework/harness/scripts/check-receipt.ts](framework/harness/scripts/check-receipt.ts)） | 与 adapter 解耦，无需动 |
| Layer 3 物理拦截 | ❌ 天然不覆盖 | 不可补；但需在共通文档里**正确表达"无 Stop hook ≠ 豁免"** |

实际问题：18 处 `framework/skills/{1-6}/SKILL.md` + 2 处 `CLAUDE.md` 散落 claude 特化措辞 `Stop hook`。SKILL.md 是 adapter 无关的共通资产，需中立化。

## 修改范围（仅文档，零脚本/JSON 改动）

### A. 6 份 SKILL.md（共 18 处替换 + 各加一段术语说明）

文件清单（括号内为 `Stop hook` 出现次数）：

- [framework/skills/1-prd-design/SKILL.md](framework/skills/1-prd-design/SKILL.md)（2）
- [framework/skills/2-requirement-design/SKILL.md](framework/skills/2-requirement-design/SKILL.md)（3）
- [framework/skills/3-coding/SKILL.md](framework/skills/3-coding/SKILL.md)（4）
- [framework/skills/4-code-review/SKILL.md](framework/skills/4-code-review/SKILL.md)（3）
- [framework/skills/5-business-ut/SKILL.md](framework/skills/5-business-ut/SKILL.md)（3）
- [framework/skills/6-device-testing/SKILL.md](framework/skills/6-device-testing/SKILL.md)（3）

**统一处理策略**：

1. 每份 SKILL.md 在「阶段闭环判定」小节首段插入一行术语说明（避免文中反复啰嗦）：

   > 下文「物理拦截层」是 adapter 中立术语：`claude` adapter 即 [.claude/hooks/check-phase-completion.mjs](framework/agents/claude/templates/hooks/check-phase-completion.mjs) 注册的 Stop hook；`generic` / `cursor` adapter 暂无等价物，仍以 Layer 1（反假设条款）+ Layer 2（完成回执 + check-receipt.ts）兜底——**没有 Stop hook ≠ 豁免 BLOCKER**，少跑一项即任务失败。

2. 文中 18 处 `Stop hook` → `物理拦截层`（其中"Stop hook 物理拦截"→"物理拦截层兜底"，避免冗余）。

### B. 实例工程下游 [CLAUDE.md](CLAUDE.md)（2 处）

- 第 138 行 `Stop hook 会读取本节定义的判据决定能否放行 stop。` → 与 [framework/templates/AGENTS.md.template](framework/templates/AGENTS.md.template) 第 138 行一致：`物理拦截层（如 Claude Code 的 Stop hook）会读取本节定义的判据决定能否放行 stop。`
- 第 148 行 `若 Stop hook 检测到任一项缺失，将以 exit code 2 阻止...` → 改为 `若物理拦截层检测到任一项缺失（claude adapter 上即 Stop hook），将以 exit code 2 阻止...`

> 改完后 [CLAUDE.md](CLAUDE.md) §5.1 / §6.5 与 [AGENTS.md.template](framework/templates/AGENTS.md.template) 渲染版本字节一致，下次 framework-init UPDATE 体检判 EMPTY，不会触发假 diff。

- [framework/agents/claude/templates/settings.json](framework/agents/claude/templates/settings.json) 与 [framework/agents/claude/templates/hooks/](framework/agents/claude/templates/hooks/check-phase-completion.mjs) — claude 专属，不动。
- [framework/skills/00-framework-init/SKILL.md](framework/skills/00-framework-init/SKILL.md) 中 2 处 `Layer 3 物理拦截` — 在描述 claude adapter 可选字段产物，措辞已正确，不动。
- [framework/templates/AGENTS.md.template](framework/templates/AGENTS.md.template) — FIX-A 已用中立措辞，不动。
