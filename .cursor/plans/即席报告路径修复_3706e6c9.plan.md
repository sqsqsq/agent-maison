## 三、已证实根因

### 3.1 唯一决定 trace/report/plan-lint 落点的代码

`[adhoc-device-test.ts](framework/harness/scripts/adhoc-device-test.ts)` L312–324：

```312:324:framework/harness/scripts/adhoc-device-test.ts
const hylyreDir = stepsFileArg
  ? path.dirname(resolveAdhocInputPath(projectRoot, stepsFileArg))
  : hylyreDirForRun(projectRoot);

const reportOutPath = path.join(hylyreDir, 'test-report.md');
const traceOutPath = path.join(hylyreDir, 'trace.json');
const lintReportPath = path.join(hylyreDir, 'plan-lint.json');
```


| 执行方式                        | `hylyreDir` 取值                                                               | 结果                           |
| --------------------------- | ---------------------------------------------------------------------------- | ---------------------------- |
| `--steps-file <path>`       | `**dirname(<path>)**`                                                        | 输出与 steps **同目录**            |
| `--plan` only（无 steps-file） | `hylyreDirForRun()` → 硬编码 `doc/features/_adhoc/testing/reports/<ts>/hylyre/` | 正确                           |
| `--observe-ui` 自动生成 steps   | L268 先 `hylyreDirForRun()`，再设 `stepsFileArg` 为其中路径                           | 正确（但 L312 仍走 dirname，恰好同一目录） |


**git 证据**：commit `6dcf0ba` 将原先**始终** `mkdir doc/features/_adhoc/.../<ts>/hylyre` 改为上述条件分支；`3994e76` 及之前无 `dirname` 分支。

### 3.2 下游写盘均使用上述三个路径（无第二套逻辑）


| 写入物                                            | 写盘位置（代码行）                                                                                                                                     |
| ---------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `plan-lint.json`                               | L356–359、L391–394；lint 失败 L156–159                                                                                                            |
| `trace.json`（占位/早期 exit）                       | `writeAdhocTracePlaceholder(traceOutPath, …)` L344+                                                                                           |
| `trace.json` + `test-report.md`（steps-file 跑通） | `device-test-run.ts` L1104–1119 `synthesizeTraceFromStepsBatchRun({ tracePath, reportPath })`，路径来自 `opts.traceOutPath` / `opts.reportOutPath` |
| `trace.json` + `test-report.md`（plan 跑通）       | `device-test-run.ts` L1257–1265 传 `--report-out` / `--trace-out` 给 hylyre                                                                     |
| `test-steps.normalized.json`                   | L148：`path.join(path.dirname(filePath), …)` — **同样跟随 steps 目录**                                                                               |


**结论**：只要 `--steps-file` 指向 `framework/harness/xxx.json`，则 `plan-lint.json`、`trace.json`、`test-report.md` **必然**落在 `framework/harness/`。  
若 steps 在 `framework/harness/reports/_adhoc/testing/<ts>/hylyre/test-steps.json`，则三者**必然**落在该 hylyre 子目录 — **与用户汇报的 legacy 路径一致，仍是 L312–314，不是独立 bug**。

### 3.3 `resolveAdhocInputPath` 使 cwd=harness 时易解析到 harness 目录

`[adhoc-input-path.ts](framework/harness/scripts/utils/adhoc-input-path.ts)` L14–18：**先** `path.resolve(process.cwd(), p)`，存在则采用。

典型命令：`cd framework/harness && npm run adhoc-device-test -- --steps-file test-steps-adhoc-wallet.json`  
→ steps 解析为 `<repo>/framework/harness/test-steps-adhoc-wallet.json`  
→ `hylyreDir = framework/harness`  
→ 三类报告写入 harness 根。

### 3.4 `test-steps-adhoc-wallet.json` 不是 CLI 写的

全仓 `grep test-steps-adhoc-wallet`：**0 匹配**。  
框架内无任何脚本生成该文件名；只能来自 **agent 手动 Write**（或用户创建）。

`lint-adhoc-steps.ts` 仅读取 `--file`，不写 steps。

### 3.5 始终走 `doc/features/_adhoc/` 的路径（与 bug 无关的部分）


| 产物                         | 路径构造                                                         | 代码                                                                                                                             |
| -------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------ |
| `derive-adhoc-last.json`   | `doc/features/_adhoc/testing/reports/derive-adhoc-last.json` | `deriveLastJsonPath()` L94–103                                                                                                 |
| `device-test-run.log/meta` | `featurePhaseReportsDir(projectRoot, '_adhoc', 'testing')`   | L206–207；本仓 `[framework.config.json](framework.config.json)` 配了 `reports_dir_pattern` → `doc/features/_adhoc/testing/reports/` |
| dump-ui 默认输出               | `doc/app-snapshot-cache/<bundle>/dump-ui-<date>.json`        | `adhoc-dump-ui.ts` L35–44                                                                                                      |


### 3.6 `framework/harness/reports/_adhoc/` **不会**被 adhoc CLI 自动创建 timestamp/hylyre

- `hylyreDirForRun()` L115–128：**硬编码** `projectRoot/doc/features/_adhoc/...`，不用 `featurePhaseReportsDir`。
- `featurePhaseReportsDir` legacy 回退（`[config.ts](framework/harness/config.ts)` L1304）仅到 `framework/harness/reports/<feature>/<phase>/`，**不含** `<timestamp>/hylyre` 子目录。
- `harness-runner --feature _adhoc`：**exit 1**，不触发 `check-testing.ts` 的 `device_test.run`（L1877 `hylyreOutDir = path.dirname(derivedPath)` 对 _adhoc 不可达）。

因此 `framework/harness/reports/_adhoc/testing/<ts>/hylyre/test-report.md` 在当前代码下**只能**由 L312 `dirname(steps-file)` 产生（steps 曾被放在该目录），**不是**另一套自动 mkdir 逻辑。

### 3.7 两套路径解析并存（审计备注，非本次主因）

同一文件内：

- **Hylyre 执行产物**：`hylyreDirForRun()` → 固定 `doc/features/...`（仅 `--plan`-only 时生效）
- **meta/log**：`featurePhaseReportsDir()` → 读 `reports_dir_pattern` 或 legacy

本仓 `projectRoot` 由 `defaultProjectRoot()` L80–85 解析：cwd 为 `framework/harness` 时上溯两级到实例根 → **meta 路径正确**。  
若误传 `--project-root` 指向无 `framework.config.json` 的目录，`DEFAULT_PATHS`（`[config.ts](framework/harness/config.ts)` L469–479）**不含** `reports_dir_pattern`，meta 会落 legacy — 但 **trace/report 仍随 steps dirname**，与主 bug 独立。
