# Init 流程体验优化 + Config 默认值调优

## 背景

分析宿主工程最新 `/framework-init` 日志 + review 反馈，发现三类问题：

**A. Agent 执行效率**：S0 npm install 后 `cd framework/harness` 失败（shell cwd 已在 harness）；S3 缺 `--smart-auto` 文档导致 2 次 CLI 试错；S4 冗余调用 `buildRunSummary`；S2 预览 Agent 自行推导与 harness 不一致（cleanup-deprecated 预览 skip 实际 run）。

**B. Config 默认值**：pypi 镜像源改华为；`module_graphs_dir` 改模块根目录；`reports_dir_pattern` 自动注入；`cross_module_exports_file` UPDATE 透传未归一化。

**C. 架构防护（review 根因）**：`ensure-config overwrite` 走 config-builder 整文件重写。核心原则：**默认值变更不得覆盖用户显式配置**——derive payload 须保留磁盘已有 `paths` / `tools`；框架默认值变更仅通过 BACKFILL（补缺）+ MIGRATION（modernize 已知旧默认）推进，不重置合法自定义值。

---

## 设计原则（review 第三轮补充）

| 操作 | 允许 | 禁止 |
|------|------|------|
| BACKFILL | 磁盘**缺失**的 key 补框架默认 | 覆盖已有 key |
| MIGRATION | modernize **已知旧默认**（如旧路径模式、旧镜像源 URL） | 覆盖合法自定义值 |
| derive payload（UPDATE overwrite） | 保留磁盘 `paths` / `tools` / `architecture` 等显式配置 | 丢弃后让 profile defaults 重置 |
| config-builder CREATE | 注入新框架默认 | — |

### 改动 7：pypi_extra_index_url 改华为镜像源（P1 修正）

**目标**：改框架**默认**镜像源，不覆盖用户已配置的自定义 URL。

**SSOT 默认值改动**（3 处）：

- [harness/config.ts](harness/config.ts) 第 1914 行
- [profiles/hmos-app/config-defaults.json](profiles/hmos-app/config-defaults.json) 第 54 行
- [templates/framework.config.template.json](templates/framework.config.template.json) 第 105 行

改为 `https://mirrors.tools.huawei.com/pypi/simple`。

**P1 风险**：当前 plan 写「overwrite 下 tools 由 profile defaults 注入，改默认即可生效」——这会在 UPDATE overwrite 时**重置**用户自定义 `tools.hylyre.pypi_extra_index_url`（例如内网源）。

**修正方案**（两层）：

1. **derive payload 保留 tools**（改动 11 扩展）：
   ```typescript
   if (existing.tools && typeof existing.tools === 'object' && !Array.isArray(existing.tools)) {
     payload.tools = JSON.parse(JSON.stringify(existing.tools));
   }
   ```
   config-builder 合成顺序 `BACKFILL → profile defaults → inputs`，inputs 中的 `tools` 优先级最高，保留用户配置。

2. **MIGRATION 规则**（P3：detect 用 type guard）：
   ```typescript
   {
     id: 'pypi_mirror_tsinghua_to_huawei',
     detect: raw => readPathString(raw, 'tools.hylyre.pypi_extra_index_url') === 'https://pypi.tuna.tsinghua.edu.cn/simple',
     // ...
   }
   ```
   仅当磁盘值**等于旧框架默认**（清华源）时迁移；其他 URL（内网源、用户自选源）不动。

**CREATE 路径**：新实例直接获得华为源默认，无影响。

### 改动 10：cross_module_exports_file 仅 canonicalize 大小写漂移（P2 修正）

**文件**：[harness/scripts/init-orchestrate.ts](harness/scripts/init-orchestrate.ts) `deriveUpdateConfigWritePayload`

**P2 风险**：上一版「按 profile 强制归一化到 index.ets / index」会**覆盖**用户显式配置的非默认导出名（如 `exports.ets`）。

**修正方案**：仅 canonicalize **已知默认的大小写漂移**，保留合法自定义值：

```typescript
const KNOWN_EXPORTS_CANONICAL: Record<string, string> = {
  'Index.ets': 'index.ets',
  'INDEX.ETS': 'index.ets',
  'Index': 'index',
  'INDEX': 'index',
};
// 仅当当前值命中已知漂移表时才替换；否则原样保留
if (typeof archClone.cross_module_exports_file === 'string') {
  const current = archClone.cross_module_exports_file;
  if (KNOWN_EXPORTS_CANONICAL[current]) {
    archClone.cross_module_exports_file = KNOWN_EXPORTS_CANONICAL[current];
  }
}
```

| 磁盘值 | 结果 | 原因 |
|--------|------|------|
| `Index.ets` | → `index.ets` | 已知默认大小写漂移 |
| `index.ets` | 不变 | 已是 SSOT |
| `exports.ets` | 不变 | 用户自定义导出名，禁止重置 |

runtime 大小写不敏感（`har-export-resolve.ts`）。未来可从 `oh-package.json5 main` 推断，不在本次范围。

### 改动 11：deriveUpdateConfigWritePayload 保留 paths + tools + 写盘模型文档

**问题根因**（review 核心）：overwrite 时 derive payload 丢弃显式配置，profile defaults 重置用户自定义值。

**代码改动**——[init-orchestrate.ts](harness/scripts/init-orchestrate.ts) `deriveUpdateConfigWritePayload` 保留：

```typescript
// paths 保留（用户自定义 paths 不被重置）

// tools 保留（P1：用户自定义 pypi 等不被 profile defaults 覆盖）

  payload.tools = JSON.parse(JSON.stringify(existing.tools));

```

**与 MIGRATION 的配合**：

| 字段 | derive 保留 | MIGRATION modernize |
|------|-------------|---------------------|
| `paths.module_graphs_dir` | 保留旧值 | 旧默认 `doc/modules/...` → 新默认 |
| `tools.hylyre.pypi_extra_index_url` | 保留任意 URL | 仅旧默认清华源 → 华为源 |
| `architecture.cross_module_exports_file` | 保留自定义名 | 仅大小写漂移 canonicalize |

**文档改动**——SKILL.md 新增「Config 写盘模型」小节：

| 场景 | 写盘路径 | 机制 |
|------|----------|------|
| CREATE | `ensure-config run` | config-builder 合成（BACKFILL 默认 + S2 payload） |
| UPDATE overwrite | `ensure-config overwrite` | config-builder 整文件重写（derive 保留显式配置 + BACKFILL 补缺） |
| UPDATE keep | 跳过 ensure-config | merge-framework-config 三 pass（backfill / migrate / confirm） |

关键约束：
- overwrite **不经过** merge-framework-config，但 overwrite 后 `backfill-config` / `migrate-config` 仍按 DAG 顺序执行
- **默认值变更不得覆盖显式配置**：derive 保留 + MIGRATION 仅 modernize 已知旧默认
