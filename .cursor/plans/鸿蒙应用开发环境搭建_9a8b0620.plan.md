# 鸿蒙原生应用开发环境搭建与真机调试全流程

## 前置条件确认

- 电脑：Windows 10/11 64位，内存 >= 16GB，磁盘剩余 >= 100GB
- 设备：华为手机，已升级至 HarmonyOS NEXT
- 账号：已有华为 ID，需升级为开发者账号
- 一根 USB 数据线（支持数据传输，非纯充电线）

---

## 第一步：安装 DevEco Studio 5.x

**目标**：安装最新版 IDE 及完整工具链（SDK、Node.js、Hvigor、OHPM 一体化打包）。

1. 访问华为开发者联盟下载页：[https://developer.huawei.com/consumer/cn/download/](https://developer.huawei.com/consumer/cn/download/)
2. 下载 **DevEco Studio 5.0.5**（或当前最新 Release 版），约 2.2GB
3. 解压后双击安装程序，选择安装目录（建议非 C 盘，路径不含中文和空格）
4. 安装过程中勾选 **DevEco Studio** 组件
5. 首次启动：
  - 同意用户协议
  - 选择 "Do not import settings"
  - 等待 SDK 和工具链自动下载完成（NEXT 版本已合一打包，开箱即用）
6. 验证安装：菜单 **Help > Diagnostic Tools > Diagnose Development Environment**，确保所有检查项为绿色通过

**关键产物**：DevEco Studio 可正常启动，SDK 路径配置完成。

---

## 第二步：注册华为开发者账号并实名认证

**目标**：将现有华为 ID 升级为开发者账号，获得 AGC 平台访问权限。

1. 浏览器访问华为开发者联盟：[https://developer.huawei.com/consumer/cn/](https://developer.huawei.com/consumer/cn/)
2. 用已有华为 ID 登录
3. 点击页面右上角 "管理中心" 或直接访问 [https://developer.huawei.com/consumer/cn/console](https://developer.huawei.com/consumer/cn/console)
4. 系统会引导你完成**开发者注册**：
  - 选择账号类型：**个人开发者**（不上架的话个人就够了）
  - 填写开发者信息（姓名、身份证号等）
  - 完成**实名认证**（人脸识别或银行卡验证，通常几分钟到1个工作日）
5. 认证通过后，访问 **AppGallery Connect**（AGC）：[https://developer.huawei.com/consumer/cn/service/josp/agc/index.html](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)
6. 确认能正常进入 AGC 控制台

**关键产物**：开发者账号状态为"已认证"，可正常访问 AGC。

> **注意**：实名认证是申请调试证书的前提，必须先完成。

---

## 第三步：创建 HarmonyOS NEXT 工程

**目标**：在 DevEco Studio 中创建一个 Stage 模型的空白工程。

1. 打开 DevEco Studio，点击 **Create Project**
2. 选择模板：**Application > Empty Ability**
3. 填写项目配置：
  - **Project name**: `SimulatedWalletForHmos`
  - **Bundle name**: `com.example.simulatedwallet`（可自定义，格式为反向域名）
  - **Save location**: `e:\1.code\SimulatedWalletForHmos`（你的现有仓库路径）
  - **Compatible SDK**: 选择 API 12 或更高
  - **Module name**: `entry`（默认）
  - **Device type**: 勾选 **Phone**
4. 点击 **Finish**，等待工程初始化和依赖下载完成

创建完成后，工程结构大致如下：

```
SimulatedWalletForHmos/
  AppScope/               # 应用级配置（app.json5）
  entry/                  # 主模块
    src/
      main/
        ets/              # ArkTS 源码
          entryability/   # UIAbility 入口
          pages/          # 页面文件
        resources/        # 资源文件
        module.json5      # 模块配置
      ohosTest/           # 测试模块（后续 UT 工作的基础）
    build-profile.json5
    oh-package.json5
  build-profile.json5     # 工程级构建配置
  oh-package.json5        # 工程级依赖管理
  hvigorfile.ts           # 构建脚本
```

**关键产物**：完整的 HarmonyOS NEXT Stage 模型工程，自带 `ohosTest` 目录（为后续模块级 UT 铺路）。

## 第四步：配置调试签名（真机调试的核心）

这里有两种方式，**推荐先用自动签名快速跑通，后续再切手动签名**。

### 方式 A：自动签名（推荐，最快）

适用于个人开发者、单设备调试场景：

1. USB 连接华为手机到电脑
2. 手机上开启 **开发者选项 > USB 调试**（设置 > 关于手机 > 连续点击版本号 7 次开启开发者模式）
3. 在 DevEco Studio 中：**File > Project Structure > Signing Configs**
4. 勾选 **Automatically generate signature**（自动签名）
5. 登录华为开发者账号（弹出登录窗口）
6. DevEco Studio 会自动完成以下操作：
  - 生成调试证书（.cer）
  - 生成 Profile 文件（.p7b）
  - 配置签名信息
7. 点击 **Apply > OK**

完成后即可直接运行调试。

### 方式 B：手动签名（完整流程，了解即可）

当自动签名不满足需求（如多设备、团队协作）时使用：

**B1. 生成密钥和 CSR 文件**

- DevEco Studio 菜单：**Build > Generate Key and CSR**
- 创建或选择 `.p12` 密钥库文件，设置密码（>=8位，含大小写+数字+特殊符号中至少两种）
- 设置密钥别名（Alias）和密码
- 填写证书信息，生成 `.csr` 文件

**B2. 在 AGC 平台申请调试证书**

- 登录 AGC > 用户与访问 > 证书管理
- 上传 `.csr` 文件，申请**调试证书**
- 下载生成的 `.cer` 证书文件

**B3. 注册调试设备**

- 手机连接电脑后，在终端执行：`hdc shell bm get --udid`
- 复制输出的 UDID
- 在 AGC > 用户与访问 > 设备管理 中添加该 UDID

**B4. 创建调试 Profile**

- 在 AGC > 用户与访问 > Profile管理 中创建
- 选择类型为 **Debug**
- 关联刚申请的调试证书和注册的设备
- 下载生成的 `.p7b` 文件

**B5. 在 DevEco Studio 中配置**

- **File > Project Structure > Signing Configs**
- 取消自动签名勾选
- 手动指定：`.p12` 文件路径、密钥库密码、密钥别名、`.cer` 文件路径、`.p7b` 文件路径

**签名所需文件汇总**：

- `.p12` — 密钥库文件（本地生成）
- `.csr` — 证书请求文件（本地生成，提交给 AGC）
- `.cer` — 调试证书（AGC 签发）
- `.p7b` — 调试 Profile（AGC 签发，绑定设备 UDID）

## 第五步：真机运行与调试

**目标**：将应用安装到手机并支持断点调试。

### 通过 DevEco Studio 直接运行（推荐）

1. USB 连接手机，确认设备被识别（工具栏设备下拉框中能看到你的手机）
2. 选择 `entry` 模块作为运行目标
3. 点击工具栏 **Run**（绿色三角）或 **Debug**（绿色虫子图标）
4. DevEco Studio 会自动：编译 > 签名 > 安装 > 启动
5. 手机上首次安装可能需要确认信任

### 通过 HDC 命令行安装（备用）

需要先配置 HDC 环境变量，将 SDK 的 `toolchains` 目录加入 PATH：

```powershell
# 添加环境变量（路径以实际 SDK 安装位置为准）
$env:PATH += ";C:\Users\<你的用户名>\AppData\Local\OpenHarmony\Sdk\<版本>\toolchains"

# 验证
hdc version

# 查看已连接设备
hdc list targets

# 获取设备 UDID（手动签名时需要）
hdc shell bm get --udid

# 安装 HAP（覆盖安装）
hdc app install -r entry\build\default\outputs\default\entry-default-signed.hap
```

### 调试能力

DevEco Studio 支持的调试方式：

- **断点调试**：在 ArkTS 代码中设置断点，Debug 模式运行
- **HiLog 日志**：通过 Log 面板查看 `console.log` / `hilog` 输出
- **热重载**：修改 UI 代码后可快速预览（部分场景支持）
- **ArkUI Inspector**：查看组件树和布局

## 第六步：构建 HAP 包（独立打包）

如果需要将 HAP 包分享给他人安装或存档：

1. 菜单：**Build > Build Hap(s)/APP(s) > Build Hap(s)**
2. 编译完成后，签名后的 HAP 文件位于：
  `entry/build/default/outputs/default/entry-default-signed.hap`
3. 此 HAP 可通过 HDC 命令安装到已注册的调试设备上

## 与后续模块级 UT 工作的衔接

本次工程搭建直接为 [模块级UT策划.md](doc/模块级UT策划.md) 中规划的工作奠定基础：

- `**ohosTest` 目录**已随工程自动创建，是后续编写 Hypium 测试用例的标准位置
- **签名和真机环境**确保 UT 不仅能在本地编译，还能在真机上执行 instrumentation 测试
- **HDC 工具链**配置完成后，可用于后续 UT 自动化执行流水线
- 工程的 Stage 模型结构（UIAbility + Page）与 DAG 中的流程节点天然对应

## 预计耗时

- 第一步 DevEco Studio 安装：约 30-60 分钟（含下载）
- 第二步 开发者注册认证：约 10 分钟 ~ 1 个工作日（取决于实名认证速度）
- 第三步 创建工程：约 5 分钟
- 第四步 配置签名：自动签名约  5 分钟 / 手动签名约 20 分钟
- 第五步 真机调试：约 5 分钟
- **总计：顺利的话 1-2 小时内可完成全部流程**
