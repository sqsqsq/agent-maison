## 二、两种入口模式

### 模式 A：Feature 需求驱动（标准流程）

```
用户：对 home-page 跑真机测试（或 /6-device-testing home-page）
       ↓
AI：加载 Skill 6 → 读 test-plan.md → 找到 15 条 TC
       ↓
AI：Step 4.5 逐条翻译 → 12 条可派生，3 条缺 selector 标跳过
       ↓
AI：写入 testing/reports/<ts>/hylyre/test-plan.hylyre.md
       ↓
AI：调 harness-runner --phase testing --feature home-page
       ↓ (build → install → ensureHylyre → hylyre run → app page save)
       ↓
AI：读 trace.json → 合成 test-report.md → 展示给用户
```

### 模式 B：即席自然语言用例（ad-hoc，不绑 feature）

```
用户："对 com.huawei.hmos.wallet，执行：打开应用，点击添加管理卡片，
       然后点击上方添加卡片，再点击非本机卡片，查看卡片列表并列出。"
       ↓
AI：识别为即席模式（无 feature 关联 / 用户明示 bundle）
       ↓
AI：将自然语言拆步骤 → 翻为 Hylyre JSON action
       ↓
AI：写入 doc/features/_adhoc/<ts>/hylyre/test-plan.hylyre.md（单 TC 或少量 TC）
       ↓
AI：直接调 hylyre run（跳过 build/install——app 已装或用户自管）
       ↓ (hylyre run → app page save 写缓存)
       ↓
AI：读 trace.json → 直接向用户报告结果（不走完整 harness 闭环）
```

**即席模式与标准模式的区别**：

| 维度 | 标准模式 | 即席模式 |
|------|---------|---------|
| 输入 | `doc/features/<feature>/test-plan.md` | 用户对话中的自然语言步骤 |
| 产物目录 | `doc/features/<feature>/testing/reports/<ts>/hylyre/` | `doc/features/_adhoc/<ts>/hylyre/` |
| build/install | harness 全流程 | 跳过（用户自管 app 已装） |
| 报告 | 合成完整 `test-report.md`，写回执、走闭环 | 仅向用户展示结果，不写 receipt |
| harness 门禁 | 全部 BLOCKER 检查 | 不走 `check-testing.ts`（直接调 `runHylyreDeviceTest`） |
| 快照缓存 | 写入 `doc/app-snapshot-cache/<bundle>/` | 同样写入（跨模式共享） |

## 三、App Snapshot Cache 自动维护

### 现状

- `doc/app-snapshot-cache/` 目录已预留（`.gitignore` 已配、config `app_snapshot_cache_dir` 已声明）
- `device-test-run.ts` 在 spawn `hylyre run` 时已注入 `HYLYRE_APP_STORE_DIR=<绝对路径>`
- **但 `hylyre run --plan` 自身不写快照**——快照由 `hylyre app page save` 单独命令产生
- 旧计划标为"后续演进"（line 939）

### 要做的

在 `device-test-run.ts` 的 `runHylyreDeviceTest` **执行 hylyre run 结束后**，追加一步：

```typescript
// hylyre run 结束后，无论成功失败，尝试保存当前页面快照供下次派生使用
spawnSync(pythonPath, ['-m', 'hylyre', 'app', 'page', 'save',
  '--bundle', bundleName,
  ...(deviceSn ? ['--device-sn', deviceSn] : [])
], {
  cwd: projectRoot,
  env: { ...process.env, HYLYRE_APP_STORE_DIR: appSnapshotCacheAbs },
  timeout: 30_000,
});
```

这样每次跑完（无论标准模式还是即席模式），都会在 `doc/app-snapshot-cache/<bundle>/pages/` 留下当前界面快照。**下次派生时 AI 可读该缓存辅助 selector 发现**。

2. **Step 4.5.2 - selector 发现**：按优先级查找：

5. **Step 4.5.5 - 落盘**：写入 `testing/reports/<ts>/hylyre/test-plan.hylyre.md`。
